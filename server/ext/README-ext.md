# WomanLive 拡張機能 (個人向け追加開発)

WomanLive 本体を**無改変**（本体ファイルは変更しない）まま、以下の機能を追加するアドオンです。
すべての追加コードは `server/ext/` 配下に閉じています。

## 仕組み（再ビルド不要・本体無改変）
- **読み込み**: 付属の `womanlive-ext.bat`（Node の `--require` プリロード）で起動すると、
  `server/index.js` を**書き換えずに**拡張をロードします
  （preload が express をフックして `require('./ext')(app)` を実行）。
  - 別法（任意）: `server/index.js` の `app.use(cors());` 直後に
    `try { require('./ext')(app); } catch (e) { console.error('[WomanLive拡張]', e); }` を
    1行追記しても有効化できます（通常の `womanlive.bat` で起動可）。両方を併用しても二重ロードはしません。
- 追加 API は `/ext/api/...`、追加 UI スクリプトは `/ext/assets/...` で配信。
- 配信時に `client/dist/index.html` へ拡張用 `<script>/<link>` を**動的に注入**（dist 自体は無改変）。
- クライアントは `fetch` をフックして「表示動画名の置換」「拡張検索」を実現し、
  React 描画後の DOM に追加 UI を差し込みます。
- 追加データは `videos.db` の**新規テーブル**（`ext_*`）に保存。既存テーブルは無改変。

## 導入と使い方
1. この `ext` フォルダを WomanLive の **`server` フォルダの中**へコピー（`server/ext` にする）。
2. `womanlive-ext.bat` を WomanLive の**ルート**（`womanlive.bat` と同じ場所）へコピー。
3. WomanLive を一度終了し、**`womanlive-ext.bat`** で起動すると拡張が有効になります。
4. 設定画面に「🧩 拡張機能 (カバー画像)」などの項目が増えています。カバー画像フォルダや DMM API ID 等を設定。
5. 各動画ページの「✎ 詳細情報を編集」から評価・出演者・品番などを登録。出演者の顔写真クリックで出演者ページへ。
   検索ページには「評価」ソートと「評価で絞込 ★1〜★5」が追加されます。

## 追加機能の一覧
記載順の概要です（各機能の詳細は以下の同名の節を参照）。
- **動画情報**: カバー画像 / 拡張メタデータ / 関連動画（メタデータ類似）/ スクリーンショット枚数表示・ソート / 表示名ソート / 検索結果カードの表示（メーカー・評価・フォルダ名）/ キーワード検索の拡張 / 検索トークン / DMM(FANZA) 商品検索 / 動画情報 一括取込
- **出演者**: 出演者ページ / 出演者一覧 / 出演者タグのプリセット / 出演者タグ 自動付与ルール / 出演者情報 一括取込
- **一覧・ナビゲーション**: シリーズ一覧 / タグ一覧 / 公開カレンダー / ブックマーク / ナビゲーション統一（ヘッダー・メニュー・アイコン）
- **操作・データ管理**: 一括操作 / バックアップ

## カバー画像
動画と同名の画像を指定フォルダから自動割り当てし、動画ページのスクリーンショット欄の先頭に表示（クリックで拡大）。
- 例: 動画 `AAA.mp4` に対し、フォルダ内の `AAA.jpg`（または .png/.webp 等）を自動表示。
- 設定画面「🧩 拡張機能 (カバー画像)」でフォルダを指定して保存。
- DMM 商品検索で取得したカバーは `ext_video_cover` テーブルに保存（フォルダ未設定でも表示可）。
- 実装: `server/ext/routes/cover.js`, `public/cover.js`, `public/settings.js`、
  API `GET /ext/api/video/:id/cover`、テーブル `ext_video_cover`。

## 拡張メタデータ
評価(★)・表示動画名・品番・公開日・シリーズ名・メーカー・レーベル・作品監督・出演者・ジャンルを、
各動画ページの「✎ 詳細情報を編集」から編集／表示します。
- 品番・シリーズ・メーカー等はクリックで同条件を検索（後述の「検索トークン」を自動付与）。評価は検索・ソート可。
- 「表示動画名」を設定すると、一覧・検索でファイル名の代わりに表示されます（fetch フックで置換）。
- 実装: `server/ext/routes/meta.js`, `public/videometa.js`、
  API `GET|PUT /ext/api/video/:id/meta`, `GET /ext/api/meta/bulk`、テーブル `ext_video_meta`。

## 関連動画（メタデータ類似・事前計算）
動画ページの「関連動画」を、拡張メタデータの一致度で選び直します（本体は無改変。
`/api/video/:id` のフックで `relatedVideos` を差し替え）。
- **本体の既定**: ファイルパスが前後する12件（実質「同フォルダ・ファイル名が近い動画」）のみで、
  メタデータを考慮しません。これを類似度スコアリングに置き換えます。
- **採点**: シリーズ／出演者／メーカー／レーベル／監督／ジャンル／タグ／公開時期／評価の一致を加重合計し、
  上位12件を返します（**シリーズ最重視**が既定）。パス近接は使いません。
- **重みは設定画面で調整可能**（設定キー `ext_related_weights`。0 でその項目を無視）。
- **事前計算**: 結果は `ext_related` テーブルに保存。動画のメタを編集すると（`ext_video_meta.updated_at`）
  その動画ぶんは自動で再計算。設定画面「🧩 関連動画（重み付け）」の「全再計算」で全動画を一括計算
  （横断的な変化＝他動画が新たに関連化するケースも反映）。
- 共有できる拡張属性が無い動画は本体のパス近傍にフォールバックします。
  「次の動画(next/prev)」は順次再生用に**パス順のまま**で変更しません。
- 実装: `server/ext/routes/related.js`, `public/settings.js`、
  API `GET /ext/api/video/:id/related`, `GET|PUT /ext/api/related/weights`,
  `POST /ext/api/related/rebuild`、テーブル `ext_related`。

## スクリーンショット枚数表示・ソート
検索結果カードに、各動画のスクリーンショット枚数を表示し、枚数で並べ替えできます。
- フォルダ名の右・ブックマークボタンの左に、単色カメラアイコン＋枚数を表示。
- 検索バーに「スクショ数」ソートを追加（押すたび 降順↓⇄昇順↑）。
- スクショは `screenshots` テーブルに `hash` で紐付くため、`hash` 一致でカウント。
- 拡張ソート（評価・スクショ数）は**キーワード検索と併用しても絞込が保たれます**
  （fetch フックの振り分けを検索語の種別ベースに統一）。
- 実装: `server/ext/routes/screenshots.js`, `public/screenshots.js`,
  `public/searchenh.js`、API `GET /ext/api/screenshots/counts`、ソート `ext_screenshots_desc|asc`。

## 表示名ソート / 名前ソートの改善
検索バーの「名前」ソートの**右隣**に「表示名」ソートを追加します（押すたび 昇順↑⇄降順↓）。
表示名(拡張の表示動画名)が未設定の動画はファイル名で並びます（本家「名前」ソートと同じ照合に統一）。
- **改善した並び順**（サーバー/クライアント共通の `ext_namekey` で統一）:
  - **ひらがな・カタカナを同一の五十音順**に統一（例: `カ → ま → ル`）。
  - **数字は桁数を考慮した自然順**（例: `1 → 2 → 10 → 100`。従来は文字コード順で `10` が `2` より前になっていた）。
  - **先頭の記号・空白は無視**し、続く文字で並べます（例: `★zzz` は `z` 扱い）。
- 全ての名前関連ソートに適用: 検索の「表示名」「名前」ソート（`routes/search.js`・`routes/fullsearch.js` の SQL 関数 `ext_namekey`）と、
  シリーズ一覧・出演者一覧・タグ一覧のクライアント側名前ソート（`WL.nameCompare`）。
- 実装: `public/namekey.js`（サーバー/クライアント共用の UMD ソートキー）、`public/searchenh.js`、
  `db.js`（SQL 関数 `ext_namekey` を登録）、`routes/search.js` / `routes/fullsearch.js`（ソート `ext_displayname_asc|desc`）。

## 検索結果カードの表示（メーカー・評価・フォルダ名）
検索結果カードの情報配置を拡張します。
- サムネ**右上にメーカー名**（半透明の白字）、サムネ**左下にフォルダ名**（動画の長さバッジと同形式）。
- もともとフォルダ名があった位置に**評価(★)**（スクリーンショット枚数・ブックマークと同じ行）。
- メーカー・評価は `GET /ext/api/meta/bulk`（戻り値に `maker` を追加）でまとめて取得・キャッシュ。
- 実装: `public/cardenh.js`、`routes/meta.js`（bulk にメーカー追加）。

## クリック検索のUI（包含 / 除外の統一）
動画ページの各値（品番・シリーズ名・メーカー・レーベル・作品監督・出演者）は、値クリックで「その値で検索」。
- 出演者にのみ付いていた包含の虫めがねは廃止し、各値のツールチップを「**「〇〇」で検索**」に統一。
- 赤い「🔍−（虫めがね＋マイナス）」は除外検索（[検索トークン](#検索トークンクリック検索で自動付与) の `@not...`）。
- 実装: `public/videometa.js`。

## キーワード検索の拡張 (表示動画名も対象)
通常のキーワード検索を、元の検索仕様(パス/タグ/各種演算子)を保ったまま「表示動画名(表示名)」も
照合対象に加えた拡張全文検索へ振り向けます。元の検索エンジンは変更していません(失敗時はそちらへフォールバック)。
- 実装: `server/ext/routes/fullsearch.js`、API `GET /ext/api/fullsearch`(クライアントの fetch フックが利用)。

## 検索トークン（クリック検索で自動付与）
**含む**: `@maker:"名前"` `@series:"名前"` `@label:"名前"` `@model:"品番"` `@director:"名前"` `@genre:"名前"`
`@performer:ID` `@rating:4` `@releasemonth:"YYYY-MM"` `@releaseyear:"YYYY"` `@release:none`
**除外（含まない）**: `@notmaker:"名前"` `@notseries:"名前"` `@notlabel:"名前"` `@notdirector:"名前"` `@notgenre:"名前"` `@notperformer:ID`
- 複数条件は AND。`@genre:"A" @notgenre:"B"` のように含む/除外を併用できます。
- **除外検索の UI**: 動画ページの各値（ジャンル/シリーズ/メーカー/レーベル/監督/出演者）の横に、
  赤い「🔍−（虫メガネ＋マイナス）」ボタンを表示。クリックでその値を**除外**して検索します
  （包含＝値テキストのクリック／通常の虫メガネ、と色・アイコンで区別）。実装は `public/videometa.js`。
- 実装: `server/ext/routes/search.js`、API `GET /ext/api/search`。

## DMM(FANZA) 商品検索によるメタデータ自動設定
各動画ページの右下「🔍」ボタンから、品番で FANZA 商品情報を検索しメタデータへ自動設定します。
1. 設定画面「🧩 DMM(FANZA) 商品検索API」に API ID とアフィリエイトID を登録（各自の DMM アカウントで取得）。
2. 動画ページで🔍を押すと、品番(model_no)で検索。品番未設定ならエラー。
3. 結果が1件ならプレビュー → 「設定する」で反映。複数なら選択 → プレビュー → 反映。0件なら通知。
4. 反映内容: title→表示動画名 / date→公開日 / genre→ジャンル / series→シリーズ名 /
   maker→メーカー / actress→出演者(自動作成) / director→作品監督 / label→レーベル。
   (品番・評価は維持。カバー画像が未設定なら imageURL.large を取得して設定)
- 検索仕様: `keyword=品番`(API仕様どおり)を先に実行し、0件なら品番→content_id 変換による
  `cid` 厳密一致でフォールバック（ハイフン付き品番は keyword ではヒットしないため）。
- 取得したカバー画像は `ext_video_cover` テーブルに保存（フォルダ未設定でも表示可。
  既存カバーがある動画は上書きしません）。
- 設定キー `ext_dmm_api_id` / `ext_dmm_affiliate_id`。実装: `server/ext/routes/dmm.js`、
  API `GET /ext/api/dmm/search`, `POST /ext/api/dmm/apply`。

## 動画情報 一括取込 (MsinMovieBrowser 形式)
設定画面の「🧩 拡張機能 (JSON からメタデータ取込)」から実行します。
- JSON を選択 →「プレビュー」(無変更で件数確認) →「取込実行」。
- 動画ファイル名(拡張子なし)と JSON の `fileName` が一致する動画にのみ設定。
- 値がある項目のみ設定。`rate":"0"` はスキップ、小数は切り上げ。
- 対応: `rate`→評価 / `tag`→既存タグ(統合) / `title`→表示動画名 / `pn`→品番 /
  `createDate`→公開日 / `series` `mfr`→メーカー `label` `director`(複数) `artist`→出演者(複数) `genre`(複数)。
- 複数値(`tag`/`artist`/`director`/`genre`)は 半角/全角スペース・カンマ・読点 で分割。
- 既定は「空欄のみ補完・タグ/出演者は統合」(非破壊)。チェックで上書きも可能。
- 実装: `server/ext/routes/import.js`、API `POST /ext/api/import/mb`。

## 出演者ページ
`/performer/:id`。氏名・ふりがな・誕生日・スリーサイズ・別名・出演者タグ等の編集、
画像アップロード＋正方形トリミング、関連動画・関連スクリーンショットを表示します。
- 動画ページの出演者の顔写真クリックで開きます。名前や🔎から同じ出演者を検索。
- 実装: `server/ext/routes/performers.js`, `public/performer.js`、
  API `/ext/api/performer/:id...`、テーブル `ext_performers`。

## 出演者一覧 (/performers)
ホーム画面の「出演者一覧」から開きます。出演者画像を大きめに横並び表示。
- ソート: 評価 / ふりがな / 身長 / 体重 / バスト / カップ数 / ウェスト / ヒップ / 年齢 (押すたび昇順⇄降順)。
- 絞り込み: 出演者タグ / 「別名・氏名が重複する出演者のみ」(同一人物の重複登録を発見)。
- 画像は最小140pxで、横幅に合わせて引き伸ばし表示(レスポンシブ。幅が広いほど列数が増える)。
- 「🗑 関連動画のない出演者を一括削除」: どの動画にも登録されていない出演者をまとめて削除
  (件数確認 → 確認の上で削除)。
- 実装: `server/ext/public/performerlist.js`、API `GET /ext/api/performers/all`,
  `POST /ext/api/performers/cleanup-unused`。

## 出演者タグのプリセット
出演者ページの「出演者タグ」を、動画タグと同様にプリセットから選んで設定できます。
出演者タグのプリセットは動画タグとは別管理です(設定キー `ext_performer_preset_tags`)。
「タグ編集...」でプリセット一覧を編集できます。
- 実装: `server/ext/public/core.js`(presetTagDialog) / `performers.js`、API `GET|PUT /ext/api/performer-tags`。

## 出演者タグ 自動付与ルール
設定画面の「🧩 出演者タグ 自動付与ルール」で、条件に合う出演者へ自動でタグを付けます。
- ルール = 条件(フィールド・演算子・値) → 付与タグ。例: 身長 ≧ 170 → 高身長。
- フィールド: 身長/体重/バスト/ウェスト/ヒップ/カップ数/評価/年齢/血液型。
  年齢は誕生日から自動算出。カップ数や血液型は文字比較(D≧C など)。
- 「適用」で全出演者へ一括付与(冪等)。出演者情報の取込時にも自動で付与されます。
- 「同期」チェック時は、条件に合わない出演者から「ルールのタグ」を外します
  (ルールに無いタグや手動タグはそのまま)。
- 設定キー `ext_performer_tag_rules`。実装: `server/ext/routes/tagrules.js`、
  API `GET|PUT /ext/api/performer-tag-rules`, `POST /ext/api/performer-tag-rules/apply`。

## 出演者情報 一括取込 (MsinMovieBrowser 形式)
設定画面の「🧩 出演者情報取込」から実行します。**先に動画メタデータ取込で出演者を作成**しておいてください。
- JSON の `name` と一致する**既存の出演者**にのみ設定（DBに無い出演者は作成しません）。
- 値がある項目のみ。`rate":"0"` はスキップ、小数は切り上げ。
- 対応: `yomi`→ふりがな / `barth`→誕生日 / `T`→身長 / `B`→バスト / `Cup`→カップ /
  `W`→ウェスト / `H`→ヒップ / `Wt`→体重 / `Blood`→血液型 / `rate`→評価 /
  `anotherName`→別名(「＝」分割・本人名は除外) / `tag`→出演者タグ(スペース分割) / `thum`→出演者画像。
- 画像はサーバーが `thum` のファイルパスを読み込み、正方形 webp に変換して登録します
  (`noimage` のものはスキップ／パスが存在しない場合は読込失敗としてカウント)。
- 既定は「空欄のみ補完・別名/タグは統合」。チェックで上書きも可能。
- 実装: `server/ext/routes/import.js`、API `POST /ext/api/import/performers`。

## シリーズ一覧 (/series)
ホーム画面の「シリーズ一覧」（出演者一覧の上）から開きます。シリーズ別に横並び表示。
- 各カード: 代表サムネイル / シリーズ名（末尾に動画本数を括弧書き）/ 平均評価（名前の下）。
- 平均評価はシリーズ内動画の評価の平均（評価0=未評価は計算から除外。全て未評価なら「評価なし」）。
- ソート: シリーズ名 / 動画本数 / 評価。
- サムネイルや名前のクリックで、そのシリーズ名で検索（`@series:"名前"`）。
- 実装: `server/ext/public/serieslist.js`、API `GET /ext/api/series`。

## タグ一覧 (/tags)
ホーム画面と右上メニューの「タグ一覧」（出演者一覧の下）から開きます。登録動画のタグをサムネイル付きで一覧表示します（UI はシリーズ一覧を踏襲）。
- 各カード: サムネイル / タグ名（サムネ下）/ 動画本数（サムネ右上の角丸バッジ）/ サムネ右下の編集ボタン。
- **既定サムネイル**はタグ名から生成した無地背景にタグ名を表示。各タグに**カスタムサムネイル**を登録すると差し替わります。
- **サムネイル編集**（出演者画像と同様の操作感）: 右下の編集ボタン →（登録済みなら拡大表示から「画像を変更／デフォルトに戻す」）→ 画像選択 → 切り取り → 登録。
  **推奨比率は 16:9（640×360）**で、サーバーが webp に整形して保存します。
- サムネイルやタグ名のクリックで、そのタグで検索（`@tag:"名前"`）。ソートはタグ名 / 動画本数。
- タグ名は大文字小文字を区別せず集計します（本体の検索/タグ保存が NOCASE のため）。
- 実装: `server/ext/public/taglist.js`、`routes/tags.js`、共通クロッパ `WL.cropDialog`(core.js)、
  API `GET /ext/api/tags`, `GET|POST|DELETE /ext/api/tag/thumb`、テーブル `ext_tag_thumb`。

## 公開カレンダー (/release-calendar)
ホーム画面の「公開カレンダー」から開きます。公開日(メタデータ)で年月別に動画をグループ化。
公開日が未設定の動画は「未分類」にまとめます。月/年/未分類をクリックでその条件の検索へ。
- 実装: `server/ext/public/releasecal.js`、API `GET /ext/api/release-calendar`。

## ブックマーク
動画をフォルダ分けしてブックマークできます。
- 追加ボタン(🔖+): 検索一覧ではフォルダ名の右、動画ページでは表示名の後に表示。ボタンの見た目で
  ブックマーク済みか分かります(済みは塗りつぶし)。
- 押すとフォルダ選択ダイアログが開き、フォルダの選択/解除・新規フォルダ追加(名前指定)ができます。
- ホーム画面の「ブックマーク」（公開カレンダーの上）からブックマークページを開きます。
- ブックマークページ: フォルダをサムネ・フォルダ名・平均評価で一覧。本数はサムネ右上の角丸バッジ。
  ソート(フォルダ名/本数/評価)、フォルダの追加・名称変更・削除が可能。
  フォルダを開く=そのフォルダの動画を検索(`@bookmark:ID`)。一覧の🔖から個別解除も可能。
- 実装: `server/ext/routes/bookmark.js`, `public/bookmark.js`, `public/bookmarklist.js`、
  API `/ext/api/bookmark/...`。テーブル `ext_bookmark_folders` / `ext_bookmarks`。

## ナビゲーション統一（ヘッダー / メニュー / アイコン）
ホーム画面の追加リンク（ブックマーク/公開カレンダー/シリーズ一覧/出演者一覧）と、
拡張ページのヘッダーを、本家のデザインに合わせて統一します。
- **アイコンの単色化**: 追加リンクと各拡張ページの見出しを、本家ホームと同じ
  単色(lucide風)アイコンへ変更（絵文字を廃止）。ブックマークは「ブックマーク追加ボタン
  から＋を除いた形」、公開カレンダーは本家「カレンダー」とは別アイコン(チェック付き)。
- **ヘッダー統一**: 拡張ページ（ブックマーク/公開カレンダー/シリーズ一覧/出演者一覧/
  出演者ページ）の上部バーを、本家ヘッダー（ロゴ/検索/操作）と同一構成へ。
  以前の「← 戻る」やページタイトルの表記は廃止。
- **メニュー(ハンバーガー)**: 本家ヘッダーの「設定(⚙)」ボタンをメニューボタン(≡)へ
  差し替え。クリックで、ホームと同じ並び順の各ページへのリンク＋最下部に「設定」を表示。
  「QRコードでアクセス」も含み、TOP以外のページからも各機能へ遷移できます。
- **設定ページは現状維持**（独自の上部バーのまま。⚙差し替えやメニューは行いません）。
- 実装: `server/ext/public/nav.js`（アイコン定義 / 共通ヘッダー `WL.pageHeader` /
  メニュー / 本家ヘッダーへの差し込み / テーマ切替の再現）/ `public/homelinks.js`（ホームのリンク）。
  本家ヘッダーは無改変で、DOM 上の「設定」リンクを実行時に置換します。

## 一括操作（検索結果で複数選択）
検索結果画面の右下に「選択モード」ボタン(チェックのFAB)が出ます。
- 選択モードボタン: 押すと選択モード開始、もう一度押すと解除。
- 選択モード中: その上に「全選択(表示中すべて)」「全選択解除」ボタン。1件以上選ぶと、さらに上に「メニュー」ボタン。
- カードをタップで選択/解除(左上にチェック、枠を強調表示)。
- メニュー: タグ追加 / 詳細情報編集 / ブックマーク追加 / 動画の削除(赤・要注意)。
  - タグ追加: プリセットから選び一括付与(既存タグは保持・重複は無視)。
  - 詳細情報編集: 入力した項目のみ上書き(空欄は変更なし)。ジャンルは既存に追加(union)。表示動画名は変更不可。
  - ブックマーク追加: 選択動画をフォルダへ一括追加(単一追加と同様)。
  - 動画の削除: 元ファイル＋関連(スクリーンショット/カバー/追加情報/ブックマーク)を削除。確認あり・取り消し不可。
- 実装: `server/ext/routes/bulk.js`, `public/bulkselect.js`、API `POST /ext/api/bulk/{tags,meta,bookmark,delete}`。
- 一覧の更新は本体の `library-updated` イベントを利用(本体は無改変)。

## バックアップ（エクスポート / インポート）
設定画面「🧩 バックアップ（追加機能データ）」から、拡張機能のデータを JSON で入出力します。
- 対象: 動画の追加メタデータ・出演者情報(画像含む)・出演者タグのプリセット・タグ付与ルール・
  カバー画像・ブックマーク・各種設定(`ext_*`)。動画ファイルやWomanLive標準データは含みません。
- エクスポート: JSON ファイルをダウンロード。
- インポート(復元): 選択した JSON で**現在の拡張データを置き換え**(確認あり)。
- 実装: `server/ext/routes/backup.js`、API `GET /ext/api/backup/export`, `POST /ext/api/backup/import`。

## 無効化・アンインストール
- **一時無効化**: `womanlive-ext.bat` ではなく通常の `womanlive.bat` で起動すれば拡張は無効になります。
  （`server/index.js` に1行追記する別法で導入した場合は、その行を削除/コメントアウトして再起動）。
- **完全削除**: `server/ext/` フォルダと `womanlive-ext.bat` を削除。保存データも消す場合は、
  設定画面のバックアップでエクスポートしておいた上で、`videos.db` の次のテーブルを DROP します:
  `ext_video_meta` / `ext_performers` / `ext_video_cover` / `ext_bookmark_folders` / `ext_bookmarks`
  （`ext_related` は再計算できるキャッシュなので任意。`settings` テーブルの `ext_` で始まるキーも拡張用の設定です）。
